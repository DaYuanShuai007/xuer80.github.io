第二章-C语言概述

2017-11-29
C

  1. C程序示例及解剖

    #include<stdio.h> 指令和头文件

    该行的作用相当于把stdio.h文件中所有的内容都输入该行所在位置，include提供了一种方便的途径共享许多程序共有的信息。

    include 这行代码是一条 C预处理器指令(preprocessor directive),通常C编译器在编译前会对源代码做一些准备工作，即预处理。

    studio.h文件包含了供编译器使用的输入和输出函数，该文件的含义是标准输入/输出头文件。头文件包含了编译器创建最终可执行程序要用到的信息，其帮助编译器把你的程序正确组合在一起。

    int main(void) 函数名

    C程序一定从main()函数开始执行，而()用于识别main()是一个函数。函数是C程序的基本模块。int是函数返回的数值类型，返回给操作系统。void表明main()不带任何参数

    注释在/和 /两个符号之间，提高程序可读性，编译器会忽略他们 //这种注释只能作用一行

    { 证明函数体开始 ， } 表示函数定义结束。

    int num; 声明，依据声明的类型，编译器为其在内存中分配存储空间

    变量命名 可以用大小写字母、数字和下划线命名，但首字母不能是数字

    声明变量的理由

    • 集中变量，方便查找，通过变量名或注释提高阅读星
    • 声明变量利于编程前规划
    • 利于发现隐藏在程序中的小错误，比如变量名拼写错误等
    • 如未声明变量，无法通过编译

    num = 1;赋值表达式

    在int num执行时，编译器在内存为变量num预留了空间，然后执行赋值语句，把值存储在之前预留的位置。

    pritf()调用函数 \n表示计算机光标移至下一行 %d告诉计算机以何种形式输出num的值

    return 0；C函数可给调用方提供的一个数值。

    提高程序可读性的技巧

    • 选择有意义的函数名
    • 写注释
    • 在函数中用空行分割概念上的多个部分

  2. 调试程序

    • 程序错误成为bug，找出错误并修正的过程叫调试debug。
    • 语法错误，语义错误
