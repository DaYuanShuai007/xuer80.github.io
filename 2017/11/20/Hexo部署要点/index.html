Hexo主题部署要点

2017-11-20
Hexo

先吐槽

​ 本网站基于Hexo(Hueman主题)+GitHub+自定义域名建设，未来可能加入七牛云用来放图。第一次用Hexo来搭建，研究了两天，期间基本把网上所有的坑都踩一遍，没有的坑自己摸索也算解决了，有的坑回头看简直无语，还好本人耐力极佳，爱好钻研，一直不放弃，最终成功，为了过坑，我差点为此去学前端语言(当然前端本来就在学习计划内)，呜呜~

​ 好吧，现在来讲述我几个很无语的坑~~

二级目录坑

​ 此坑甚深，为了在hueman主题增加二级目录，遍寻网页其他hueman主题的帖子，找不到答案，为此还自己修改失败，导致重新加载hexo，可怜啊，最后发现，只需要categories: [一级目录,二级目录,三级目录]即可，我简直蠢。。。

​ 不，都怪Hueman作者啊，这样自动集成功能，只字未提。。。不，不能怪我蠢，都是作者的问题。。。

兼容坑

​ 这是最大的坑，搭建网站期间，需要用到git，node以及相关插件，需要适配相关的环境变量等等，稍有差池。。重新来过。。。。这只是吐槽。。。

删除文章

​ 删除文章只需要删除blog/source/_post目录下相关文章即可

空格坑

​ 站点部署时，_congfig-yml中须在url中填写自己的域名或者yourname.github.io地址，由于语法问题，文件中所有冒号：后必须空一格。

域名坑

​ 由于本地文件上传后都会被解析成html文件，最好在本地blog/source文件夹下，添加CNAME格式文件，不是txt格式哦，里面写上自己的域名即可，如xuer80.com，写www.xuer80.com可能会限制搜索范围。

deployer-git坑(待解决)

​ 这个坑很大，应该是插件问题。在用npm install hexo-deployer-git –save后，这个插件时而会失效，一旦失效后果很严重，知乎网上都有说删除deployer-git文件重新下，不顶用，没效果，即使成功也是只把public中的文件上传到git。

npm install hexo-deployer-git

然(⊙﹏⊙)，这根本不是deployer 的错，是npm的锅(接好。。)，因为npm install之后也是这个问题，一喵一样的，更新npm之后(npm i npm -g)也是这样的。。。。

rm -rf .deploy_git之后再hexo d，时而管用，时而。。。好吧，我没管用过

  • 终于解决了，正确的做法，如果hexo d之后，最后没有显示INFO Deploy done: git，那么部署是不成功的，这个时候删除.deploy_git，然后再hexo -g d就行了，只是hexo d的话，只会提交public文件夹下的内容，网站不会更新。现在回头看，好。。。算了，，不吐槽了

    ​

插件坑

​ 总之，少装插件

关评论

​ 由于Hueman主题默认disqus评论，这是非常坑的评论系统，需要翻墙(你需要lantern)，然而这并不是关闭它的理由，因为他在评论下方会推荐其他五湖四海的博客，相当于广告。。。。自己枕边岂留她人酣睡！！！，找到themes/hueman/layout/comment文件夹，这里面是内置的评论系统布局文件，找到disqus.ejs,打开，找到dsq.src = '//' + disqus_shortname + '.disqus.com/<% if (page.comments) { %>embed.js<% } else { %>count.js<% } %>这行代码，前面加//注释掉，这样无论你在墙内还是墙外，整个世界都干净了

文章内错误导致的无法提交

之前博客提交都正常，突然一个文章写完后提交不正常了，显示什么模板错误之类的，可能是因为文章内的符号导致的。本人之前在写数组时，多维数组的嵌套，会导致提交失败，还有正则表达式，实验之后也是失败，所以谨慎使用，修改之后就可以提交了

重装系统后如何继续写作

由PC换了台式机，可博客有时还是要写，那该如何继续呢

首先把原硬盘上的博客文件夹拷贝，把其中的git和node的安装文件删除，然后，重新安装git和nodejs，配置环境变量。然后重新生成密钥，把公钥拷贝到github上保存。git要设置邮箱和登录名，然后安装hexo，npm无响应，可以安装来自淘宝的源cnpm，效果一样，之后就是cnpm install,以及deployer-git安装，完毕后既可以继续写作了
