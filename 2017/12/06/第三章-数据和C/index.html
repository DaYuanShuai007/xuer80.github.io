第三章-数据和C

2017-12-06
C

  1. 变量与常量数据

    • 数据就是承载信息的数字和字符
    • 有些数据类型在程序使用之前已预先设定，并在整个程序运行过程中没有变化，即为常量，constant
    • 其他数据类型在程序运行期间可能会改变或被赋值，称为变量。

  2. 数据：数据类型关键字

    C语言中，最初有int long short unsigned char float double 7个关键字，C90加了signed,void关键字，C99添加了_Bool , _Complex, _Imaginary三个关键字。

    计算机存储方式分两大基本类型：整数类型和浮点类型

    位、字节和字是描述计算机数据单元或存储单元的术语

    最小的存储单元是位bit，可存储0或1。位是计算机内存的基本构建块

    字节byte是常用的计算机存储单位。一字节均为8位(至少衡量存储单位时是这样)，那么8位直接就有256(2的8次方)种可能的0、1组合。通过二进制编码，可表示0~255的整数或一组字符。

    字是设计计算机时给定的自然存储单位，如现在计算机的32位，64位等，计算机的字长越大，其数据转移越快，允许的内存访问也更多

    • 计算机以二进制数字存储整数，如存7，二进制是111，存在8位字节中，需要把前5位都设置为0，后三位设置成1.
    • 一个值后面加小数点就成为了浮点值。计算机把浮点数分成小数部分和指数部分两部分存储。浮点数可表示的范围比整数大。
    • 因为任何区间内都有无数个实数，所以计算机浮点数不能表示区间所有值，浮点数只能近似于实际值。

  3. C语言基本数据类型

    1. int类型

      C语言中的整型类型可表示不同的取值范围和正负值。int类型是符号整型，其取值范围依计算机系统而异。一般而言，存储一个int要占用一个机器字长，如个人计算机32位，那么用32位来存储一个int值。ISO C规定int取值范围最小为-32768~32767。

      • 声明int变量

        int a;

        int a,b,c;

        以上均是有效声明

      • 初始化变量

        初始化变量即为变量赋一个初始值。

        int a = 21;

        int b =22,c = 23;

        =为赋值符号。

        声明类型，为变量创建和标记存储空间，初始化，则为其内存空间赋值。

      • int类型常量

        22,23等都是整型常量或整型字面量。C把不含小数点和指数的数作为整数。非常大的整数除外，可能是long类型。 %d(转换说明)与int值匹配，可以使int类型的常量、变量或表达式

      • 八进制和十六进制

        因为8和16都是2的幂，所以很常用。如十六进制3是0011,5是0101。35的位组合是00110101,53的组合是01010011.这种对应关系使得十六进制和二进制转换很方便。用0X或0x前缀表示十六进制。单独的0表示八进制。当然无论采用哪种进制表示，计算机内部都以二进制进行编码。十进制显示数字用%d；以八进制显示数字用%o；以十六进制显示数字用%x。如果要显示其前缀，必须使用%#o,%#0X。

    2. 其他整数类型

      C语言提供3个附属关键字修饰基本整数类型：short、long和unsigned。

      • short int类型(简写short) 占用存储空间可能比int类型少，常用于较小数值以节约空间，与int类似，其有符合类型
      • long int 或long 占用的存储空间可能比int多，用于较大数值场合，也是有符号类型。
      • long long int 或long long(C99标准加入)，占用的存储空间可能比long多，适用于更大数值场合。该类型至少占64位。也是有符号类型
      • unsigned int 或 unsigned只用于非负值的场合。这种类型与有符号类型表示的范围不同，如16位unsigned int允许的取值范围是0~65535，而不是-32768~32767.因此无符号整数可表示更大的数。
      • C90标准中，添加了unsigned long int或unsigned long、unsigned short int或unsigned short类型，C99标准又添加了unsigned long long int 或unsigned long long。
      • 在任何有符号类型前面添加关键字signed，可强调使用有符号类型的意图。无符号类型一定要加unsigned以区分。

      为何short类型可能比iint类型占用的空间少，long类型可能比int类型占用的空间多？因为C语言只规定short占用的存储空间不能多于int，long占用的存储空间不能少于int。这样规定是为了适应不同的机器。

      C标准对基本数据类型只规定了允许的最小值。如16位机，short和int最小取值范围是[-32768,32767]，32位机，long的最小取值也有规定。

      int类型，首先应用考虑unsigned类型，通常用于计数，因为不用负值，可以表示更大的正数。

      一般程序的数值默认为int类型。更大的会根据实际情况选择不同的类型。

      整数溢出 ：即整数超出了相应类型的取值范围。比如对于 int和unsigned int超过最大值时，变量会从最小值重新计数，不同的是unsigned从0开始重新计数，int则从负数最小值开始计数。当然C标准为定义有符号类型的溢出规则，还应注意其他情况。

      打印输出：unsigned int类型，用%u转换说明；long %ld；%lx表示十六进制打印long类型；%lo表示八进制打印long类型。

      对于short类型，可用h前缀。%hd表示十进制显示short；%ho 八进制显示；%lu

      用错误的转换会得到意外的结果。

      为何要进行转换：int类型被认为是计算机处理整数类型时最高效的类型，因此在short和int大小不同的计算系统中，用int类型的参数传递速度更快。

      h修饰符啥用？使用h修饰符可以显示较大整数被截断成short类型的情况。

      so,在使用printf时，检查转换说明与类型匹配很重要。

    3. 使用字符：char类型

      char类型用于存储字符(如字母或标点符号)，但从技术层面看，char是整数类型，因为char实际上存储的是整数而不是字符。计算机使用数字编码来处理字符，即用特定的整数表示特定的字符，本书用ASCII编码，其中整数65代表A,因此存储字母实际上存储的是整数65。

      标准ASCII码的范围是0~127，只需7位二进制数即可表示。一般char类型被定义为8位的存储单元，容纳ASCII码绰绰有余。

      C语言把1字节定义为char类型占用的位bit数，因此无论是多少位的系统，都可以使用char类型。

      • 声明

        char re;

        char a,b;

      • 字符常量和初始化

        char re = ‘A’; C中单引号括起来的字符被称为字符常量，编译器一发现’A’，就会将其转换成相应的代码值。

        单引号只适用于字符、数字和标点符号，有一些ASCII字符打印不出。C提供3种方式表示

        如使用ASCII码 char beep = 7; 蜂鸣字符的ASCII码值是7

        使用转移序列 PS:把转移序列赋给字符变量时，必须用单引号把转移序列括起来。

        如char ne = ‘\n’表示换行。C标准规定警报字符不得改变光标位置。其余字符如分页，换行等字符主要是对活跃位置或光标位置的影响。

      • 转义序列和ASCII码之间，转义序列更具可移植性，优先使用

      • %c指明打印的字符。如果用%d则会打印相应的整数值。

      • char如果为有符号类型，则取值-128~127，无符号则0~255.根据C90标准，C允许char前用signed或unsigned修饰。这在处理小整数时很有用，如果用来处理字符，那么char前就无需使用任何修饰符。

    4. _Bool类型

      C99标准添加_Bool类型，用于表示布尔值，即逻辑值true和false。C用1表示true，0表示false。所以 _Bool类型实际上也是一种整数类型。但原则上仅占1位存储空间，因为对0和1而言，1位存储空间足够了。

    5. 可移植类型：stdint.h和inttypes.h

      没看懂

    6. float、double和long double

      浮点类型是包括小数在内更大范围的数，类似于科学计数法，常表示非常大或非常小的数，有float、double和long double类型。如322.56，科学计数表示3.2256x10^2，指数计数法3.2256e2。

      C标准规定，float类型必须至少能表示6位有效数字。通常系统存储一个浮点数要占用32位，其中8位用于表示指数的值和符号，剩下24位用于表示非指数部分(也叫作尾数或有效数)及其符号。

      C提供的另一种浮点类型是double(双精度)，其和float类型的最小最值范围相同，但至少必须能表示10位有效数字。一般double占用64位，系统将多的32位全部用来表示非指数部分，不仅增加了有效数字的位数(即提高了精度),而且还减少了舍入误差。无论哪种分配方式，double类型的值至少有13位有效数字，超过了标准的最低位数规定。

      C语言规定的第三种浮点类型是long double，以满足比double类型更高的精度要求。不过C只能保证其至少与double类型的精度相同。

      • 浮点型常量

        有符号的数字 -1.23E+12 2.87-3。

        正号可省略，小数点和指数部分不能同时省略。如 3.13 .2 4e16 .8E-5 100.

        编辑器默认浮点型是double类型精度，在浮点数后加后缀f，即可看成float类型运算。加L则表示long double类型。

      • C99添加新的浮点常量格式：用十六进制表示浮点型常量，即十六进制数前加十六制前缀0x，用p代表e，用2的幂代替10的幂，如0xa.1fp10

        十六进制a等于十进制10,.lf是1/16加上15/256，p10是2^10或1024。

      • 打印 用%f转换说明表示打印十进制计数法的float和double类型浮点数，%e打印指数计数法的浮点数。如系统支持十六进制格式浮点数，可用a和A代替e和E。打印long double类型要使用%lf、%le或%la转换说明。

      • 浮点值的上溢和下溢

        当计算导致数字过大，超过当前类型表达范围，就会发生上溢。C规定，这种情况下会给其赋一个表示无穷大的值。

        当除以一个很小的数时则较为复杂。float类型的数以指数和尾数部分存储。计算过程中损失原末尾有效位上数字的情况称为下溢。C把损失了类型全精度的浮点值称为低于正常的浮点值。C库已提供了用于检查计算是否会产生低于正常值的函数。
