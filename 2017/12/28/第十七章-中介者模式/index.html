屠龙技设计模式

第十七章-中介者模式

2017-12-28
设计模式

一、介绍

中介者模式(Mediator Pattern)是对象的行为型模式，也称为调解者模式或调停者模式。这种模式提供一个中介类，该类通常处理不同类之间的通信，并支持松耦合，用来降低多个对象和类之间的通信复杂性。

二、定义

用一个中介者对象封装一系列的对象交互，中介者使各对象不需要显示的相互作用，从而使耦合松散，而且可以独立地改变他们之间的交互。

中介者模式使多对多的相互作用转化为一对多的相互作用，并将对象的行为和协作抽象化。

三、应用场景

  • 当对象之间交互操作频繁，且对象间严重依赖，为防止修改一个对象的行为同时又涉及其他很多对象的行为或修改时，可用中介模式来解决紧耦合问题。即多类相互耦合，形成网状结构。该模式将对象多对多的关系变成一对多的关系，将系统从网状结构变成以中介者对象为中心的星状结构，达到降低系统复杂性提高扩展性的作用
  • 欲通过一个中间类来封装多个类中的行为，而又不想生成太多的子类

四、分析

角色介绍：

Mediator:抽象中介者角色，定义了同事类对象到中介者对象的接口，用于各个同事之间通信，主要方法为一个或多个事件方法。

ConcreteMediator:具体中介者角色，继承于抽象中介者，实现了父类定义的方法，其知晓所有具体同事类，并负责协调各同事对象的交互关系。从具体的同事对象接收消息，向具体的同事对象发出命令。

Colleague:抽象同事类角色，定义了中介者对象的接口，它只知道中介者而不知道其他的同事对象

ConcreteColleagueA/B：具体同事类角色，继承于抽象同事类，实现自己的方法，在需要与其他同事通信时，就与持有它的中介者通信。每个具体同事类都知道本身在小范围内的行为，而不知道它在大范围内的目的。一般同事类有多个组成，他们之间互相影响，相互依赖，关系复杂。引入中介者模式，同事类之间可以从网状结构变成星状结构，任何一个类的变动只会影响自身和中介者，这样就减少了系统耦合，通过中介者来管理不属于自己的行为。,中介者模式中，同事类之间必须通过中介者才能进行消息传递

五、总结

面向对象编程中，类与类之间必然产生联系，如依赖如网状复杂，适当使用中介者模式进行解耦会使逻辑更加清晰。

优点：

  • 解耦

    中介者模式把多个同事对象间的交互封装到中介者对象中，从而使同事对象间松耦合，基本做到互不依赖，如此同事对象就可以独立变化和复用。

  • 集中控制交互

    多个同事对象的交互，被封装在中介者对象里面集中管理，在交互发生变化时，只需修改中介者类就可以了，也可以扩展修改，而不需要更改众多的同事类

  • 多对多变成了一对多，符合迪米特原则

    使对象关系更清晰

缺点

  • 过度集中化会导致中介者类非常复杂，而且难以维护
