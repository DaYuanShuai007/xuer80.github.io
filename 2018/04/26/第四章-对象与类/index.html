第四章-对象与类

2018-04-26
Java

  1. 面向对象程序设计概述

    ​ 面向对象的程序是由对象组成的，每个对象包含对用户公开和隐藏的实现部分。传统结构化设计是通过先确定算法，然后考虑存储数据的方式来设计。这也是算法+数据结构=程序的经典设计方案。而OOP将数据放在第一位，然后考虑数据的算法

    1. 类

      ​ 类是构造对象的模版或蓝图，由类构造对象的过程成为创建类的实例instance。

      ​ 封装可以将对象的数据和行为组合在一起，并对调用者隐藏数据的实现方式。对象中的数据成为实例域，操纵数据的过程称为方法。无论何时，只要向对象发送消息，对象的实例域或者状态就会改变。

      ​ 封装的关键在于，绝不能让类中的方法直接访问其他类的实例域。程序仅通过对象的方法与对象的数据进行交互。这意味着再操作方法流程不变的情况下，类可以改变数据的存储方式

    2. 对象的三个特性

      ​ 对象的行为——是由可调用的方法定义的，以为对象可以做那些操作

      ​ 对象的状态——即对象的特征信息，这会随时间改变，但不是自发的，必须通过调用方法实现。如果没掉用方法就改变了对象状态，那么说明封装失败

      ​ 对象的标识——即识别相同行为和状态的对象的标识，每个对象都有唯一的身份，作为类的实例，主要区别就在于其方法和状态不同，二者又相互影响

    3. 识别类

      ​ 找所谓的名词和动词，名词多为状态，动词多为方法

    4. 类之间的关系

      • 依赖 uses-a 即一个类的方法操纵另一个类的对象，就是一个类依赖另一个类
      • 聚合 has-a 即一个类的对象包含另一个类的对象
      • 继承 is-a 即一个类扩展另外一个类

  2. 使用预定义类

    ​ 不是所有的类都具有面向对象的特征，如Math类，只封装了功能，并没有数据，因此不会实例化

    1. 对象与对象变量

      ​ 在Java程序设计中，使用构造器constructor构造新实例。构造器是一种特殊的方法，用来构造并初始化对象。构造器的名字与类名相同，如

      new Cat()

      这个表达式构造了一个新对象，并初始化对象的相关特性。但这样对象并不方便使用，如果可以更方便的多次使用，需要将对象存放再一个变量中：

      Cat blueCat = new Cat()

      需要注意的是，只是Cat redCat，对象变量redCat可以引用Cat类型的对象，但变量redCat不是一个对象，实际上也没有引用对象，贸然调用Cat的方法将产生编译错误。必须首先初始化，可以让新构造的对象初始化该变量 redCat = new Cat()，也可让这个变量引用一个已存在的对象redCat = blueCat;如此两个变量引用同一对象。

      一定要认识到：一个对象变量并没有实际包含一个对象，而仅仅引用一个对象

      在Java中，任何对象变量的值都是对存储再另外一个地方的一个对象的引用。new操作符的返回值也是一个引用。

      如 Cat blueCat = new Cat()

      表达式new Cat()构造了一个Cat类型的对象，并且它的值时对新创建对象的引用。这个引用存储再变量blueCat中。

      局部变量不会自动地初始化为null，必须通过调用new或设置为null进行初始化

    2. Java类库中的LocalDate类

      Java标准库中有个表示时间点的Data类，其有一个状态，即特定时间点。时间使用距离一个固定时间点(纪元UTC时间1970-01-01 00：00：00)的毫秒(可正可负)表示的。还有各表示日历表示法的LocalDate类，两个类主要是把时间与日历分开。

      不要用构造器来构造LocalData类对象，可以静态工厂方法代表调用构造器

      LocalDate.new() 会创建一个新对象，表示构造这个对象时的日期

      当然，可以将构造的对象保存再一个对象变量中

      LocalDate newYear = LocalDate.of(1999,12,31)

      一旦有了LocalDate对象，可以用方法getYear，getMonthValue和getDayOfMonth得到年月日

      LocalDate类封装了实例域来维护所设置的日期，不看源码时不知道类内部的日期表示的。封装的意义就在于此，当然最重要的时对外提供的方法。

    3. 更改器方法与访问器方法

      只访问对象而不修改对象的方法成为访问器方法，如get方法

      改变对象状态或数据的方法就是更改器方法，如add,set等方法

  3. 用户自定义类

    ​ 只有一个类有main方法

    1. 在一个源文件中(包含main方法的类)，只能有一个公有类，但可以有任意数目的非公有类

    2. 类定义解析

      class people{

      private String name;

      private int age;

      public people(String n,int a){

      name = n;

      age = a;

      }

      public String getName(){

      return name;

      }

      public int getAge(){

      return age;

      }

      }

      方法关键字public意味着任何类的任何方法都可以调用这些方法，而属性关键字private确保只有people类自身的方法能够访问，而且他的类的方法不能够读写,如果用public则破坏了封装。

      由构造方法看出，构造器与类同名，构造people对象时，构造器会运行，实例化对象状态。构造器与方法的不同在于，其总伴随着new操作符的执行被调用，而不能对一个已经存在的对象调用构造器来达到重新设置类属性的目的

      每个类都可以有一个以上的构造器，每个构造器可以有不止0个参数。

      方法用于操作对象以及存取它们的属性实例域。方法中有隐式参数和显式参数，隐式参数时出现再方法名前的类对象，这称为方法调用的目标或接受者。方法括号中的参数就是显式参数。

      Java中，所有的方法都必须再类的内部定义，但并不表示它们时内联方法。

    3. 封装的优点

      分析getName和getAge方法，都是典型的访问器方法，只返回实例域值

      一个较好的获取和设置类实例域的值，只需满足三个条件

      • 一个私有的属性数据域
      • 一个公有的域访问器方法，如get
      • 一个公有的更改器方法，如set

    4. 方法的设计

      方法可以公有，也可私有，私有的方法，不会被外部的其他类操作调用，甚至可以删除，但如果方法时公有的，就不能删除，因为可能有其他代码调用

    5. final实例域或属性

      如果将实例域或属性定义为final，那么构建对象时必须初始化这样的域，且再以后的操作中，不能对其修改。该修饰符多用于基本属性或不可变的属性设置

  4. 静态域与静态方法

    1. 静态域(静态变量)

      private static String name;

      如果将域定义为static,每个类只有一个这样的域。这样的静态域将被该类的每一个对象共享。因为静态域属于类，不属于单独的对象

    2. 静态常量

      如Math.PI，PI就是一个静态常量，是Math类的一个实例域，需要用Math类来访问PI，每一个Math对象都有其单独的PI拷贝

    3. 静态方法

      静态方法是不能由对象操作的方法，只能类来操作

      如Math.pow()，可认为静态方法时没有this参数的方法

      静态方法可以访问自身类中的静态域

      以下两种情况建议使用静态方法

      • 一个方法不需要访问对象状态，其所需参数都是通过显示参数提供
      • 一个方法只需要访问类的静态域

      static修饰的域为属于类不属于类对象的变量和函数

    4. 工厂方法

      静态方法的另外一种用途就是类似LocalDate和NumberFormat的类使用静态工厂方法来构造对象。

      那么为何不利用构造器实例化对象呢？

      • 无法命名构造器。构造器名字必须与类名相同，但是这里想得到的两个实例用不同的名字
      • 当使用构造器时，无法改变所构造的对象类型。而静态方法返回的是NumberFormat的子类

    5. main方法

      main方法也是一个静态方法，但不对任何对象进行操作。程序启动时还没有任何一个对象。静态的main方法将执行并创建程序所需要的对象。

  5. 方法参数

    ​ Java总是采用按值调用和传递的，方法得到的是所有参数值的一个拷贝，尤其，方法不能修改传递给它的任何参数变量的内容。

    ​ 方法参数共有两个类型：基本数据类型和对象引用

    ​ 一个方法不可能修改一个基本数据类型的参数，但可以改变一个对象参数的状态。因为方法得到的是对象引用的拷贝，对象引用及其他的拷贝同时引用同一个对象。一个方法不能让对象参数引用一个新的对象。

    ​ C++有值调用和引用调用，Java只有值调用

  6. 对象构造

    由于对象构造非常重要，Java提供了多种编写构造器的机制

    1. 重载

      类可以有多个构造器 如下

      多个方法有相同的名字，不同的参数，便是重载。编译器通过各个方法给出的参数类型与特定方法调用所使用的值类型进行匹配来挑选出相应的方法。如找不到匹配的参数，就会产生编译时的错误，这个过程成为i额重载解析。

      Java允许重载任何方法，而不只是构造器方法。因此，要完整地描述一个方法，需要指出方法名以及参数类型，这称为方法的签名。

      返回类型不是方法签名的一部分，因此不能有两个名字、参数类型相同但返回类型不同的方法。

    2. 默认域初始化

      如果构造器没有显式地给域赋予初值，那么就会自动地赋为默认值：数值为0、布尔值为false、对象引用为null。如果不明确地对域进行初始化，会影响代码的可读性，开发中应避免。

      方法中的局部变量必须明确初始化，而类中的域，如未初始化，将自动初始化为默认值

    3. 无参数的构造器

      很多类都包含一个无参数的构造函数，对象由无参构造函数创建时，其状态会设置为适当的默认值。

      如果一个类没有编写任何构造器，那么系统就会提供一个无参构造器，其实例也设置为默认值。如果一个类提供仅提供一个有参构造器，而没有无参构造器时，系统并不会自动创建无参构造器，这样创建对象时，调用无参构造器将产生错误。

      所以，仅当类没有提供任何构造器时，系统才提供一个默认的构造器。如果系统所有域被赋予默认值，可采用以下形式

    4. 显式域初始化

      通过重载类的构造器方法，可采用多种形式设置类的实例域的初始状态。确保不管怎样调用构造器，每个实例域都可以被设置为一个有意义的初值。

      在执行构造器前，先执行赋值操作。当一个类的所有构造器都希望把相同的值赋予某个特定的实例域时，这种方式非常有用

      初始值不一定是常量值，也可以调用方法进行初始化。

    5. 参数名

      参数变量可用与实例域同样的名字，但可以用this.实例域名字 这种形式访问实例域，因为this指隐式参数，也就是所构造的对象。

    6. 调用另一个构造器

      如果用this(…..)的形式，这个构造器将调用同一个类的另一个构造器。采用这种this关键字非常有用，这样对公共的构造器代码部分，只编写一次即可

      当调用people(String)时，people(Int age,name)也一同被调用。

    7. 初始化块

      • 在构造器中设置值
      • 在声明中赋值

      实际上，Java还有初始化块，在一个类的声明中，可包含多个代码块，只要构造类的对象，这些块就会被执行

      如上，无论哪个构造器构造对象，sum域都再对象初始化块中被初始化，因为首先运行初始化块，然后才运行构造器的主体部分。不过应避免这种操作，通常可以直接将初始化代码放在构造器中。

      如果把一个代码块，加上关键字static，那么在类第一次加载时，将会对静态域进行初始化。

    8. 对象析构与finalize方法

      Java有自动的垃圾回收器，不需要人工回收内存，不支持析构器。可以为任何一个类添加finalize方法，其将再垃圾回收器清楚对象之前调用。但这个方法并不一定有效，因为并不确定它什么时候才能被调用。

  7. 包

    ​ Java允许使用包package将类组织起来，借助包可以方便管理代码，并将自己的代码和别人提供的代码分开管理

    ​ 使用包的主要原因是确保类名的唯一性，两个同名类，放再不同的包中就可以避免冲突。一般包的命名用公司域名倒置，如com.baidu.www形式。每一个包都是一个类集合

    1. 类的导入

      一个类可以使用所属包中的所有类，以及其他包中的公有类。

      访问另一个包中的公有类，有两种方式，一，在每个类名前添加完整的包名，很繁琐；另一种是用import语句，这是一种引用包含再包中的类的简述，一旦使用了Import语句，在使用类时，就不必写出包的全名了。

      在包中定位类时编译器的工作，如有不同包但名称相同的类，可用完整的包名来区分。

    2. 静态导入

      import不仅可以导入类，还可导入静态方法和静态域的功能，如

      就可以使用System类的静态方法和静态域，而不必加类名前缀

    3. 将类放入包中

      如果想把一个类放入特定的包中，可以在定义类的代码前，将包的名字放在源文件开头，否则就默认放在默认包中。

    4. 包作用域

      修饰符，标记为public的部分可以被任意类访问；标记为private的部分只能被定义它们的类使用；如果没有指定，那么这个部分(类、方法或变量)可以被同一个包的所有方法访问

  8. 类路径

    ​ 类存储再文件系统的子目录中，类的路径必须与包名匹配

    ​ 另外类也可以存在Jar(Java归档)文件中。在一个Jar文件中，可包含多个压缩形式的类文件和子目录，这样既可以节省空间又可以改善性能。

    ​ 为了使类能被多个程序共享，可做以下几点

    • 把类放到包树状结构的基目录中
    • 将Jar文件放在一个目录中
    • 设置类路径class path。类路径是所有包含类文件的路径的集合

  9. 文档注释

    JDK包含一个Javadoc的工具，可由源文件生成一个html文档。在源代码中以/**注释，可以将代码与注释保存再一个地方。由于文档注释与源代码再同一个文件中，在修改源代码的同时，重新运行javadoc就可以轻易保持二者一致。

    1. 注释的插入

      javadoc实用程序从以下特征中抽取信息

      • 包
      • 公有类与接口
      • 公有的和受保护的构造器及方法
      • 公有的和受保护的域

      以上及部分可编写注释，注释应放置在这些部分前面。并以/*开始， /结束。

      每个/*… /文档注释在标记之后，紧跟着自由格式文本，由@开始，如@author或@param

    2. 类注释

      类注释必须放在import语句之后，类定义之前

    3. 方法注释

      每一个方法注释必须放在所描述的方法之前。除了通用标记之外，还可以使用下面标记

      @param变量描述 一个方法的所有@param标记必须放一起，可占据多行

      @return描述 标记返回部分，可跨约多行

      @throws 描述可能抛出的异常

    5. 通用注释

      下面的标记可用在类文档的注释中

      @author姓名 可以采用多个标记，每个标记对应一个作者

      @version文本 该标记是对当前版本的描述

      @since文本 该标记会产生一个since条目，可用于所有的文档注释中，这里的text可以是对引入特性的版本描述，如@since version 1.0

      @see引用 这个标记将在see also部分增加一个超链接，可用于类中，也可用于方法中

    6. 包与概述注释

      可直接将类、方法和变量的注释放置再Java源文件中，用/*… /文档注释界定就可以了，但包注释，就需要在每一个包目录中添加一个单独的文件。可以有两个选择

      • 提供一个以包.html命名的html文件，里面的…文本都会抽取出来
      • 提供一个以package-info.java命名的文件，内部必须包括/*和 /界定的javadoc注释，跟随再一个包语句之后

    7. 注释的抽取

      如果是一个包，应运行命令

      javadoc -d docDirectory 包名

      如果多个包生成文档，运行

      javadoc -d docDirectory 包1 包2

      如果文件在默认包中，运行

      javadoc -d docDirectory *.java

  10. 类设计技巧

    • 一定要保证数据私有

      绝对不要破坏封装性，最好保证实例域的私有

    • 一定要对数据初始化

      Java不对局部变量初始化，但是会对对象的实例域进行初始化。不要依赖系统的默认值，而是应显式初始化所有的数据

    • 不要再类中使用过多的基本类型

      可以用其他类代替多个相关的基本类型的使用，这样更易于理解和修改

    • 不是所有的域都需要独立的域访问器和更改器

    • 将职责过多的类进行分解

    • 类名和方法名要能够体现职责

    • 优先使用不可变类

      比如LocalDate包中的其他类是不可变的，没有方法能修改对象的状态，这样可以安全地再多个线程共享其对象。如果多线程同时更新一个对象，可能发生并发更改
